# Дизайн-ревью: карточка отпуска / техспринта в свимлейне

## Назначение

Карточка показывает в свимлейне спринт-планера периоды **отпуска** и **техспринтов** (Web / Back / QA) по данным из квартального планирования. Элемент только для отображения: его нельзя перетаскивать и удалять из спринт-планера (редактирование — в квартальном планировании).

---

## Текущая реализация

- **Компонент:** `features/swimlane/components/AvailabilityCard.tsx`
- **Слой:** `AvailabilityCardsLayer` рендерит карточки поверх сетки дней в каждом свимлейне.
- **Стили:**
  - `rounded-lg`, `border-2` — в одну систему с карточками задач
  - `shadow-sm` — лёгкая тень без hover (неинтерактивный элемент)
  - Фон в dark: `/70` (например `dark:bg-emerald-900/70`) для читаемости
  - Шрифт: 11px, `fontWeight: 600`, подпись по центру карточки
- **Поведение:** текст прокручивается вместе с карточкой (нет sticky/fixed).

---

## Что получилось хорошо

1. **Чёткое отличие от задач**  
   Один компактный блок с подписью, без ключа, SP и тегов — сразу видно, что это не задача и что элемент фиксирован.

2. **Вписывание в сетку**  
   Позиция и ширина привязаны к дням (как у баров задач), колонки и выравнивание не ломаются.

3. **Семантика «неинтерактивно»**  
   Нет hover-тени и ручки перетаскивания, блок выглядит статичным.

4. **Цветовая кодировка**  
   Отпуск (янтарь), Back (изумруд), Web (голубой), QA (янтарь) — в одну палитру с командами и квартальным планированием.

5. **Тёмная тема**  
   Отдельные тёмные варианты, блок не теряется на тёмном фоне.

---

## Решения и ограничения

### Текст при скролле

- **Идея:** оставлять подпись на месте при горизонтальном скролле (sticky/fixed).
- **Почему не делаем:**  
  - `position: sticky` не работает внутри родителя с `position: absolute` (карточка в сетке — absolute).  
  - Вариант с отдельным слоем подписей и скролл-контейнером усложняет разметку и поддержку.  
  - Вариант с `position: fixed` и обновлением `top` по скроллу добавляет ref, scroll listener и обновление позиции — больше кода и рисков рассинхрона.  
- **Текущее решение:** текст остаётся внутри карточки и прокручивается вместе с ней — предсказуемо и без костылей.

## Рекомендации на будущее

1. **Единообразие с TaskCard**  
   При изменении скругления, толщины рамки или тени у карточек задач — синхронизировать с `AvailabilityCard` (сейчас уже: `rounded-lg`, `border-2`, `shadow-sm`).

2. **Доступность**  
   У карточки есть `title={config.label}` — подсказка при наведении; при необходимости можно добавить `aria-label` для скринридеров.

3. **Узкие карточки**  
   При длине в 1 день подпись может обрезаться (`truncate`). При желании можно добавить тултип по hover или чуть уменьшить размер шрифта для очень узких карточек (сейчас не делаем).

4. **Не трогать**  
   Вертикальные отступы (`verticalMargin = 6`) и размер шрифта (11px, 600) согласованы с TaskBar/карточками задач — менять без веской причины не стоит.

## Итог

Карточка отпуска/техспринта по смыслу и месту в интерфейсе вписана удачно: однозначно отличается от задач, не выглядит перетаскиваемой и логично сидит в сетке по исполнителям и дням. Визуальное единство с карточками задач обеспечено (скругление, рамка, тень, плотность фона в dark). Текст при скролле намеренно оставлен «вместе с карточкой», без sticky/fixed, чтобы не усложнять реализацию.
