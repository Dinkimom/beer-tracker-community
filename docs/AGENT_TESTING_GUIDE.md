# Руководство по тестированию для агентов (MCP)

## Обзор проекта

**Beer Tracker** — это веб-приложение для планирования спринтов в Яндекс Трекере по свимлейнам. Приложение позволяет визуально планировать задачи на 10 рабочих дней, распределяя их между разработчиками и тестировщиками.

### Основные возможности

1. **Планирование спринтов** — визуальное распределение задач по дням и разработчикам
2. **Управление бэклогом** — просмотр и управление задачами, не включенными в спринт
3. **Планирование фич** — создание и управление фичами с задачами
4. **Burndown chart** — визуализация прогресса спринта
5. **Цели спринта** — управление целями через чеклисты

### Технологический стек

- **Frontend**: Next.js 16, React 19, TypeScript
- **Backend**: Next.js API Routes
- **База данных**: PostgreSQL
- **Внешние API**: 
  - Yandex Tracker API (для изменения данных и работы со спринтами)
  - PostgreSQL `issue_snapshots` (для чтения данных о задачах, бэклоге, стори)
- **UI библиотеки**: Tailwind CSS, Radix UI, dnd-kit (drag-and-drop)

## Архитектура приложения

### Структура страниц

Приложение имеет две основные страницы:

1. **Главная страница** (`/`) — основная рабочая область
   - Вкладка "Спринты" с подвкладками:
     - **Бэклог** — список задач, не включенных в спринт
     - **Планировщик** — визуальное планирование задач по свимлейнам
     - **Burndown** — график выгорания спринта
   - Вкладка "Фичи" — планирование фич

2. **Выбор доски** (`/select-board`) — выбор доски при первом запуске

### Основные компоненты

#### 1. SprintPlanner (Планировщик спринта)
- Отображает свимлейны для каждого разработчика/тестировщика
- Каждый свимлейн разделен на 10 рабочих дней, каждый день — на 3 части
- Задачи можно перетаскивать между свимлейнами и днями
- Задачи можно растягивать/уменьшать по времени

#### 2. Sidebar (Боковая панель)
- Список задач спринта
- Бэклог задач
- Цели спринта (чеклист)
- Настройки разработчиков
- Метрики спринта

#### 3. BacklogPage (Страница бэклога)
- Список всех задач, не включенных в спринт
- Возможность перетаскивания задач в спринты

#### 4. Feature Planner (Планировщик фич)
- Список фич
- Редактор фичи с задачами
- Документация фичи
- Связи между задачами

## Пользовательские истории и тест-кейсы

### 1. Выбор доски

**Пользовательская история**: Как пользователь, я хочу выбрать доску для работы, чтобы видеть спринты и задачи этой доски.

**Тест-кейсы**:

1. **TC-001: Первый запуск приложения**
   - **Шаги**:
     1. Открыть приложение
     2. Проверить, что отображается страница выбора доски
     3. Проверить, что отображаются все доступные доски (Booking, Порог входа, Вертикали, Operations)
   - **Ожидаемый результат**: Страница выбора доски отображается с 4 досками

2. **TC-002: Выбор доски**
   - **Шаги**:
     1. На странице выбора доски кликнуть на доску "Booking"
     2. Дождаться загрузки
   - **Ожидаемый результат**: 
     - Приложение перенаправляет на главную страницу
     - В шапке отображается название выбранной доски
     - Загружаются спринты для выбранной доски

3. **TC-003: Смена доски**
   - **Шаги**:
     1. На главной странице кликнуть на селектор доски в шапке
     2. Выбрать другую доску
   - **Ожидаемый результат**: 
     - Загружаются спринты для новой доски
     - Выбранный спринт обновляется (выбирается активный или первый в списке)

---

### 2. Работа со спринтами

**Пользовательская история**: Как пользователь, я хочу видеть список спринтов и выбирать активный спринт для планирования.

**Тест-кейсы**:

1. **TC-010: Отображение списка спринтов**
   - **Шаги**:
     1. Открыть главную страницу
     2. Перейти на вкладку "Спринты" → "Планировщик"
   - **Ожидаемый результат**: 
     - Отображается селектор спринтов
     - В списке видны все спринты доски (активные, завершенные, будущие)

2. **TC-011: Выбор спринта**
   - **Шаги**:
     1. В селекторе спринтов выбрать другой спринт
   - **Ожидаемый результат**: 
     - Загружаются задачи выбранного спринта
     - Отображаются свимлейны с задачами
     - Обновляется боковая панель с задачами спринта

3. **TC-012: Автоматический выбор активного спринта**
   - **Шаги**:
     1. Выбрать доску
     2. Открыть планировщик
   - **Ожидаемый результат**: 
     - Автоматически выбирается спринт со статусом "in_progress"
     - Если активного спринта нет, выбирается первый в списке

---

### 3. Планирование задач в свимлейнах

**Пользовательская история**: Как пользователь, я хочу перетаскивать задачи из боковой панели на свимлейны, чтобы планировать работу разработчиков.

**Тест-кейсы**:

1. **TC-020: Отображение свимлейнов**
   - **Шаги**:
     1. Открыть планировщик спринта
   - **Ожидаемый результат**: 
     - Отображаются свимлейны для каждого разработчика/тестировщика
     - Каждый свимлейн разделен на 10 рабочих дней
     - Каждый день разделен на 3 части (утро, день, вечер)

2. **TC-021: Перетаскивание задачи из сайдбара на свимлейн**
   - **Шаги**:
     1. В боковой панели найти задачу
     2. Перетащить задачу на нужный день и часть в свимлейне разработчика
   - **Ожидаемый результат**: 
     - Задача появляется на свимлейне в указанном месте
     - Задача исчезает из боковой панели (или остается, если это копия)
     - Позиция задачи сохраняется

3. **TC-022: Перемещение задачи между свимлейнами**
   - **Шаги**:
     1. Перетащить задачу из одного свимлейна в другой
   - **Ожидаемый результат**: 
     - Задача перемещается на новый свимлейн
     - Позиция задачи обновляется

4. **TC-023: Перемещение задачи между днями**
   - **Шаги**:
     1. Перетащить задачу на другой день в том же свимлейне
   - **Ожидаемый результат**: 
     - Задача перемещается на новый день
     - Позиция задачи обновляется

5. **TC-024: Изменение размера задачи**
   - **Шаги**:
     1. Навести курсор на правый край задачи
     2. Потянуть вправо или влево
   - **Ожидаемый результат**: 
     - Длительность задачи изменяется
     - Задача не выходит за границы 10 дней

6. **TC-025: Возврат задачи в сайдбар**
   - **Шаги**:
     1. Перетащить задачу из свимлейна обратно в сайдбар (в область "Неназначенные задачи")
   - **Ожидаемый результат**: 
     - Задача удаляется из свимлейна
     - Позиция задачи удаляется
     - Задача появляется в сайдбаре

### 4. Работа с бэклогом

**Пользовательская история**: Как пользователь, я хочу видеть задачи бэклога и добавлять их в спринты.

**Тест-кейсы**:

1. **TC-030: Отображение бэклога**
   - **Шаги**:
     1. Перейти на вкладку "Спринты" → "Бэклог"
   - **Ожидаемый результат**: 
     - Отображается список задач, не включенных в спринты
     - Задачи сгруппированы по спринтам или статусам

2. **TC-031: Перетаскивание задачи из бэклога в спринт**
   - **Шаги**:
     1. В бэклоге найти задачу
     2. Перетащить задачу в колонку активного спринта
   - **Ожидаемый результат**: 
     - Задача добавляется в спринт
     - Задача исчезает из бэклога

3. **TC-032: Перетаскивание задачи из спринта в бэклог**
   - **Шаги**:
     1. В планировщике найти задачу
     2. Перетащить задачу в колонку бэклога
   - **Ожидаемый результат**: 
     - Задача удаляется из спринта
     - Задача появляется в бэклоге

### 5. Цели спринта

**Пользовательская история**: Как пользователь, я хочу управлять целями спринта через чеклист.

**Тест-кейсы**:

1. **TC-040: Отображение целей спринта**
   - **Шаги**:
     1. Открыть планировщик спринта
     2. Открыть боковую панель
     3. Перейти на вкладку "Цели"
   - **Ожидаемый результат**: 
     - Отображается список целей спринта (чеклист)
     - Отображается прогресс (например, "3/5 выполнено")

2. **TC-041: Отметка цели как выполненной**
   - **Шаги**:
     1. В списке целей кликнуть на чекбокс цели
   - **Ожидаемый результат**: 
     - Чекбокс отмечается как выполненный
     - Прогресс обновляется
     - Изменения сохраняются в Tracker

3. **TC-042: Добавление новой цели**
   - **Шаги**:
     1. В списке целей кликнуть на кнопку "Добавить цель"
     2. Ввести текст цели
     3. Сохранить
   - **Ожидаемый результат**: 
     - Новая цель добавляется в список
     - Цель сохраняется в Tracker

4. **TC-043: Редактирование цели**
   - **Шаги**:
     1. Кликнуть на текст цели
     2. Изменить текст
     3. Сохранить
   - **Ожидаемый результат**: 
     - Текст цели обновляется
     - Изменения сохраняются в Tracker

5. **TC-044: Удаление цели**
   - **Шаги**:
     1. Кликнуть на кнопку удаления цели
     2. Подтвердить удаление
   - **Ожидаемый результат**: 
     - Цель удаляется из списка
     - Изменения сохраняются в Tracker

### 6. Работа с задачами

**Пользовательская история**: Как пользователь, я хочу видеть информацию о задаче и изменять её статус.

**Тест-кейсы**:

1. **TC-050: Отображение информации о задаче**
   - **Шаги**:
     1. Кликнуть на задачу в планировщике или сайдбаре
   - **Ожидаемый результат**: 
     - Открывается модальное окно или сайдбар с информацией о задаче
     - Отображается название, описание, статус, исполнитель, story points, test points

2. **TC-051: Изменение статуса задачи**
   - **Шаги**:
     1. Открыть задачу
     2. Кликнуть правой кнопкой мыши на задачу
     3. Выбрать "Изменить статус" → выбрать новый статус
   - **Ожидаемый результат**: 
     - Статус задачи обновляется
     - Изменения сохраняются в Tracker
     - Задача обновляется в интерфейсе

3. **TC-052: Обновление story points**
   - **Шаги**:
     1. Открыть задачу
     2. Изменить значение story points
   - **Ожидаемый результат**: 
     - Story points обновляются
     - Изменения сохраняются в Tracker

4. **TC-053: Обновление test points**
   - **Шаги**:
     1. Открыть задачу
     2. Изменить значение test points
   - **Ожидаемый результат**: 
     - Test points обновляются
     - Изменения сохраняются в Tracker

### 7. Планирование фич

**Пользовательская история**: Как пользователь, я хочу создавать фичи и планировать задачи для них.

**Тест-кейсы**:

1. **TC-060: Отображение списка фич**
   - **Шаги**:
     1. Перейти на вкладку "Фичи"
   - **Ожидаемый результат**: 
     - Отображается список всех фич
     - Для каждой фичи отображается название, статус, прогресс

2. **TC-061: Создание новой фичи**
   - **Шаги**:
     1. На странице списка фич кликнуть на кнопку "Добавить фичу"
     2. Ввести название фичи
     3. Сохранить
   - **Ожидаемый результат**: 
     - Создается новая фича
     - Открывается страница редактирования фичи

3. **TC-062: Добавление задачи в фичу**
   - **Шаги**:
     1. На странице редактирования фичи кликнуть на кнопку "Добавить задачу"
     2. Ввести название задачи
   - **Ожидаемый результат**: 
     - Задача добавляется в фичу
     - Задача отображается на странице

4. **TC-063: Редактирование задачи в фиче**
   - **Шаги**:
     1. Кликнуть на задачу в фиче
     2. Изменить название или другие параметры
   - **Ожидаемый результат**: 
     - Изменения сохраняются
     - Задача обновляется в интерфейсе

5. **TC-064: Добавление тегов к задаче**
   - **Шаги**:
     1. Кликнуть правой кнопкой мыши на задачу в фиче
     2. Выбрать "Теги" → выбрать тег (Web, Back, QA)
   - **Ожидаемый результат**: 
     - Тег добавляется к задаче
     - Тег отображается на карточке задачи

6. **TC-065: Добавление story points к задаче**
   - **Шаги**:
     1. Кликнуть правой кнопкой мыши на задачу в фиче
     2. Выбрать "Story Points" → выбрать оценку (1, 2, 3, 5, 8)
   - **Ожидаемый результат**: 
     - Story points добавляются к задаче
     - Оценка отображается на карточке задачи

7. **TC-066: Создание связи между задачами**
   - **Шаги**:
     1. Навести курсор на задачу в фиче
     2. Кликнуть на кнопку добавления связанной задачи
     3. Ввести название связанной задачи
   - **Ожидаемый результат**: 
     - Создается новая задача, связанная с исходной
     - Между задачами отображается стрелка связи

**Пользовательская история**: Как пользователь, я хочу видеть график выгорания спринта.

**Тест-кейсы**:

1. **TC-070: Отображение графика выгорания**

## Критерии успешного тестирования

### Функциональные требования

- ✅ Все задачи можно перетаскивать между свимлейнами и днями
- ✅ Позиции задач сохраняются после перезагрузки страницы
- ✅ Изменения статусов задач синхронизируются с Tracker
- ✅ Цели спринта отображаются и редактируются корректно
- ✅ Бэклог отображает задачи, не включенные в спринты
- ✅ Фичи создаются и редактируются корректно

### Нефункциональные требования

- ✅ Приложение загружается менее чем за 3 секунды
- ✅ Перетаскивание задач работает плавно (без лагов)
- ✅ Интерфейс адаптивен (работает на разных размерах экрана)
- ✅ Ошибки обрабатываются корректно (показываются пользователю)

## Известные проблемы

Список известных проблем находится в файле `BUGS_AND_TASKS.md`:

1. Не находится задача на цели спринта, хотя она есть
2. Закрывается сайдбар в спринт планере по клику вне сайдбара, нужно убрать это поведение
3. При попытке создания диаграммы происходит ошибка
4. При открытии задач/стори отображаются только драфты, а задачи/стори из трекера не отображаются
5. Уменьшение карточки в спринт планере слева уменьшает карточку справа

## Инструкции для агентов MCP

### Подготовка к тестированию

1. **Запуск приложения**:
   ```bash
   pnpm dev
   ```
   Приложение будет доступно по адресу `http://localhost:3000`

2. **Проверка окружения**:
   - Убедитесь, что настроены переменные окружения (`.env.local`)
   - Проверьте подключение к базе данных PostgreSQL
   - Проверьте доступность Tracker API

### Процесс тестирования

1. **Навигация по приложению**:
   - Используйте браузер через MCP Playwright для навигации
   - Делайте скриншоты на каждом этапе тестирования
   - Проверяйте консоль браузера на наличие ошибок

2. **Выполнение тест-кейсов**:
   - Выполняйте тест-кейсы последовательно
   - Для каждого тест-кейса:
     - Выполните шаги
     - Проверьте ожидаемый результат
     - Зафиксируйте результат (✅ пройден / ❌ провален)
     - Если тест провален, сделайте скриншот и опишите проблему

3. **Проверка интеграций**:
   - Проверьте, что данные синхронизируются с Tracker API
   - Проверьте, что снимки задач загружаются из PostgreSQL
   - Проверьте, что позиции задач сохраняются в PostgreSQL
