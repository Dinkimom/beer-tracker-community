# Квартальное планирование

## Две версии (v1 и v2)

- **v2 (актуальная):** `features/quarterly-planning-v2/`, страница через `QuarterlyPlanningV2Page`. Над этой версией ведётся разработка.
- **v1 (legacy):** `features/quarterly-planning/`. Старая реализация; часть UI/утилит ещё импортируется из v2. Не является целью для новых фич.

**Стратегия:** вся новая функциональность — в v2 (или в `lib/` при общем использовании). Правила импортов и направление зависимостей: **[ADR-001-quarterly-planning-v1-v2.md](./ADR-001-quarterly-planning-v1-v2.md)**. Дополнительно: `features/quarterly-planning/README.md`.

**API:** общие роуты в `app/api/quarterly-plans/`; отдельный роут v2 — `app/api/quarterly-plans/v2/route.ts`. Клиент — `lib/api/quarterly.ts` (и старые, и v2-методы).

## Обзор

Модуль квартального планирования позволяет планировать эпики, стори и задачи на квартал вперед с разбивкой по спринтам или неделям. Поддерживается создание драфтов задач и планирование фаз разработки.

## Основные возможности

### 1. Временная шкала
- **По спринтам**: отображение по 2-недельным спринтам
- **По неделям**: отображение по неделям квартала

### 2. Планирование задач
- Добавление эпиков, стори и задач из трекера
- Создание драфтов задач (еще не созданных в трекере)
- Drag & Drop из сайдбара в timeline
- Удаление задач из плана

### 3. Фазы разработки
Для каждой задачи можно указать фазы:
- **Discovery** (Дискавери) - фиолетовый
- **Development** (Разработка) - синий
- **Testing** (Тестирование) - желтый
- **Release** (Релиз) - зеленый

### 4. Фильтрация
- Поиск по названию
- Фильтр по типу (epic/story/task/draft)
- Фильтр по статусу
- Фильтр по тегам/платформам
- Показывать только незапланированные

## Структура данных

### Типы
См. `types/quarterly.ts`:
- `QuarterlyPlan` - квартальный план
- `PlannedItem` - запланированный элемент
- `TaskPhase` - фаза задачи
- `DraftTask` - драфт задачи

### База данных
Таблицы (миграция `009_quarterly_planning.sql`):
- `quarterly_plans` - планы по кварталам
- `planned_items` - запланированные элементы
- `draft_tasks` - драфты задач

## API Endpoints

### Квартальные планы
```
GET  /api/quarterly-plans?boardId=X&year=Y&quarter=Z
POST /api/quarterly-plans
```

### Драфты
```
GET  /api/drafts?boardId=X
POST /api/drafts
```

## Использование

### Открытие страницы
Перейдите на вкладку "Квартальное планирование" в главном меню или используйте URL:
```
/?page=quarterly
```

### Создание плана
1. Выберите квартал и год
2. Перетащите задачи из сайдбара в timeline
3. Добавьте фазы, кликнув на пустые ячейки
4. Настройте даты и типы фаз

### Создание драфта
1. Нажмите "Создать драфт" в сайдбаре
2. Заполните форму
3. Драфт появится в списке доступных задач
4. Позже можно создать задачу в трекере

## Troubleshooting

### Задачи не отображаются
- Проверьте применена ли миграция БД
- Проверьте подключение к API
- Откройте консоль браузера для ошибок

### Фазы не сохраняются
- Убедитесь, что даты валидны
- Проверьте наличие таблицы `planned_items`

### Драфты не создаются
- Проверьте права на запись в БД
- Убедитесь, что `boardId` передан корректно
