# Возможности приложения Beer Tracker

## Обзор

**Beer Tracker** — это веб-приложение для планирования и управления спринтами в Яндекс Трекере. Приложение предоставляет визуальный интерфейс для распределения задач по дням и разработчикам, управления бэклогом, планирования фич и квартального планирования.

## Основные возможности

### 1. Планирование спринтов

Визуальное планирование задач на 10 рабочих дней с распределением между разработчиками и тестировщиками.

**Функции:**
- **Свимлейны** — отдельная дорожка для каждого разработчика/тестировщика
- **Временная сетка** — каждый день разделен на 3 части (утро, день, вечер)
- **Drag & Drop** — перетаскивание задач между свимлейнами и днями
- **Изменение длительности** — растягивание/уменьшение задач по времени
- **Автоматическая расстановка** — автоматическое распределение задач по разработчикам
- **Сохранение позиций** — позиции задач сохраняются в базе данных

**Компоненты:**
- `SprintPlanner` — главный контейнер планировщика
- `Swimlane` — дорожка для разработчика
- `TaskCard` — карточка задачи
- `Sidebar` — боковая панель с задачами

### 2. Управление бэклогом

Просмотр и управление задачами, не включенными в спринты.

**Функции:**
- **Список задач** — отображение всех задач бэклога
- **Фильтрация** — по статусу, типу, исполнителю, тегам
- **Поиск** — поиск задач по названию или ключу
- **Группировка** — по спринтам, статусам, типам
- **Добавление в спринт** — перетаскивание задач из бэклога в спринт
- **Пагинация** — постраничная навигация

**Компоненты:**
- `BacklogPage` — страница бэклога
- `BacklogTab` — вкладка бэклога в сайдбаре

### 3. Планирование фич

Создание и управление фичами с задачами, документацией и связями.

**Функции:**
- **Список фич** — просмотр всех фич проекта
- **Создание фич** — добавление новых фич
- **Редактор фич** — редактирование задач, тегов, оценок
- **Документация** — добавление документации к фичам
- **Связи между задачами** — визуальное отображение зависимостей
- **Чеклисты качества** — автоматическая проверка декомпозиции
- **Расчет сроков** — автоматический расчет сроков разработки (3 SP = 1 день, 3 TP = 1 день)
- **Планирование релизов** — управление релизами фич
- **Диаграммы** — создание и редактирование диаграмм (Excalidraw)

**Компоненты:**
- `FeaturePlanner` — планировщик фич
- `FeatureTabs` — вкладки фичи
- `FeatureDocumentEditor` — редактор документации
- `FeatureDiagramEditor` — редактор диаграмм

### 4. Квартальное планирование

Планирование эпиков, стори и задач на квартал вперед с разбивкой по спринтам или неделям.

**Функции:**
- **Временная шкала** — отображение по спринтам (2 недели) или по неделям
- **Иерархия задач** — поддержка структуры Epic → Story → Task с визуальным отображением
- **Планирование задач** — добавление эпиков, стори, задач из трекера
- **Драфты задач** — создание драфтов задач (еще не созданных в трекере) с поддержкой иерархии (эпики, стори, задачи)
- **Фазы разработки** — планирование фаз с указанием частей дня (утро, день, вечер):
  - **Discovery** (Дискавери) — фиолетовый
  - **Development** (Разработка) — синий
  - **Testing** (Тестирование) — желтый
  - **Release** (Релиз) — зеленый
- **Автоматическое планирование** — автоматическое распределение задач с учетом:
  - Зависимостей между задачами (топологическая сортировка)
  - Доступности разработчиков (техспринты, отпуска, загрузка)
  - Платформ задач (back/web/qa)
  - Story Points и Test Points
  - Capacity команды (30 SP/TP на спринт)
  - SLA-багов (стабильное планирование из спринта в спринт)
  - Ресурса на ошибки планирования
- **Управление доступностью** — учет техспринтов и отпусков:
  - Техспринты по платформам (Back, Web, QA)
  - Отпуска участников команды
  - Визуальное отображение в timeline
- **Управление участниками** — выбор участников для квартального планирования:
  - Участники текущей команды
  - Поиск участников из других команд
  - Сохранение состава участников на квартал
- **Drag & Drop** — перетаскивание задач из сайдбара в timeline
- **Редактирование фаз** — добавление, изменение и удаление фаз через модальное окно
- **Связи между задачами** — визуальное отображение зависимостей между задачами
- **Фильтрация** — по типу (epic/story/task/draft), статусу, тегам, платформам, незапланированным задачам
- **Поиск** — поиск задач по названию или ключу
- **Группировка** — группировка задач по спринтам в сайдбаре
- **Загрузка дочерних задач** — автоматическая загрузка дочерних задач для эпиков и стори
- **Автоматическая сортировка** — сортировка задач стори по порядку
- **Экспорт в Excel** — экспорт плана в Excel с сохранением структуры, фаз и форматирования
- **Навигация** — переключение между кварталами (Q1-Q4) и годами
- **Оптимистичные обновления** — мгновенное отображение изменений без ожидания ответа сервера

**Компоненты:**
~`QuarterlyPlanningPage` — главная страница~
- `TimelineGrid` — таблица с временной шкалой
- `TasksSidebar` — сайдбар с задачами
- `PhaseCell` — ячейка с фазами
- `PhaseEditModal` — модальное окно редактирования фазы
- `DraftTaskModal` — модальное окно создания/редактирования драфта
- `AutoPlanModal` — модальное окно автоматического планирования
- `AvailabilityEditModal` — модальное окно управления доступностью (техспринты/отпуска)
- `ParticipantsModal` — модальное окно управления участниками
- `QuarterSelector` — выбор квартала/года
- `QuarterlyPlanningHeader` — заголовок с настройками и действиями

### 5. Burndown Chart

Визуализация прогресса спринта через график выгорания.

**Функции:**
- **График выгорания** — отображение идеального и фактического выгорания
- **Метрики** — общее количество SP/TP, оставшиеся SP/TP
- **Переключение спринтов** — просмотр графика для разных спринтов
- **Интерактивность** — наведение на точки графика для деталей

**Компоненты:**
- `BurndownChart` — график выгорания
- `BurndownMetrics` — метрики спринта

### 6. Цели спринта

Управление целями спринта через чеклисты.

**Функции:**
- **Чеклист целей** — список целей спринта
- **Добавление целей** — создание новых целей
- **Редактирование** — изменение текста целей
- **Отметка выполнения** — отметка целей как выполненных
- **Удаление** — удаление целей
- **Прогресс** — отображение прогресса выполнения (например, "3/5 выполнено")
- **Синхронизация** — сохранение в Tracker API

**Компоненты:**
- `GoalsTab` — вкладка целей в сайдбаре
- `GoalItem` — элемент цели
- `SprintGoalsModal` — модальное окно управления целями

### 7. Управление задачами

Работа с задачами: просмотр, редактирование, изменение статусов.

**Функции:**
- **Просмотр задачи** — детальная информация о задаче
- **Изменение статуса** — переходы между статусами через контекстное меню
- **Обновление оценок** — изменение story points и test points
- **Создание задач** — создание новых задач в трекере
- **Связанные задачи** — создание связанных задач
- **Комментарии** — добавление и просмотр комментариев
- **История изменений** — просмотр changelog задачи
- **Связи между задачами** — визуальное отображение связей

**Компоненты:**
- `TaskCard` — карточка задачи
- `TaskModal` — модальное окно задачи
- `ContextMenu` — контекстное меню задачи
- `CommentCard` — карточка комментария

### 8. Управление разработчиками

Настройка и управление участниками команды.

**Функции:**
- **Список разработчиков** — отображение всех участников команды
- **Участники команды** — получение участников из PostgreSQL приложения (team_members/staff)
- **Мердж данных** — объединение участников команды с исполнителями из спринта
- **Роли** — определение ролей (developer/tester) на основе role_slug из team_members
- **Настройки** — управление видимостью разработчиков в планировщике

**Компоненты:**
- `DevelopersManagement` — управление разработчиками
- `DeveloperHeader` — заголовок свимлейна разработчика

### 9. Выбор доски

Выбор доски для работы при первом запуске.

**Функции:**
- **Список досок** — отображение всех доступных досок
- **Выбор доски** — выбор доски для работы
- **Смена доски** — переключение между досками в шапке
- **Сохранение выбора** — сохранение выбранной доски в localStorage

**Компоненты:**
- `BoardSelector` — селектор доски
- `SelectBoardPage` — страница выбора доски

### 10. Учет работы

Учет работы разработчиков по задачам.

**Функции:**
- **Модальное окно учета** — ввод времени работы
- **Связь с задачами** — привязка учета к задачам спринта

**Компоненты:**
- `AccountWorkModal` — модальное окно учета работы

## Технические возможности

### Интеграции

1. **Yandex Tracker API**
   - Получение спринтов
   - Изменение данных (создание, обновление, удаление задач)
   - Работа с чеклистами
   - Управление статусами

2. **PostgreSQL (issue_snapshots)**
   - Получение задач (issues)
   - Получение бэклога
   - Получение стори и эпиков
   - Получение истории изменений (changelog)
   - Получение возможных переходов статусов

3. **PostgreSQL приложения (team_members/staff)**
   - Получение участников команды
   - Получение ролей сотрудников
   - Мердж с исполнителями из спринта

4. **PostgreSQL (локальная БД)**
   - Сохранение позиций задач
   - Сохранение связей между задачами
   - Сохранение комментариев
   - Хранение фич и документации
   - Хранение квартальных планов

### Безопасность

- **Пользовательские токены** — каждый пользователь работает под своим OAuth токеном
- **Хранение токенов** — токены хранятся только в браузере (localStorage)
- **HTTPS** — все запросы идут через HTTPS
- **Fallback токен** — поддержка серверного токена как fallback

### Производительность

- **Connection Pooling** — пул соединений для PostgreSQL
- **Оптимизированные запросы** — использование JOIN'ов
- **Кэширование** — возможность кэширования результатов
- **Graceful Degradation** — при ошибках интеграций приложение продолжает работать

## Архитектура

### Feature-Based структура

Приложение организовано по принципу **domain-driven design**:
- Каждая фича изолирована со своими компонентами, хуками и утилитами
- Shared код вынесен в общие папки
- Легко добавлять новые фичи
- Легко удалить или заменить фичу

### Основные домены

- `sprint/` — планирование спринтов
- `task/` — работа с задачами
- `sidebar/` — боковая панель
- `swimlane/` — свимлейны
- `backlog/` — бэклог
- `burndown/` — график выгорания
- `feature-planner/` — планирование фич
- `quarterly-planning/` — квартальное планирование
- `developers/` — управление разработчиками
- `qa/` — QA логика
- `board/` — выбор доски
- `context-menu/` — контекстное меню
- `account/` — учет работы
- `comments/` — комментарии

## API

### Архитектура API

- **Клиент** → `beerTrackerApi` (axios) → Next.js API Routes
- **Сервер** → маршруты `/api/*`: Yandex Tracker API, PostgreSQL (`lib/db`, `lib/snapshots/*`, tenant-таблицы); без ClickHouse
- **База данных** → PostgreSQL (приложение и снимки задач)

### Основные эндпоинты

- `/api/sprints` — работа со спринтами
- `/api/tracker` — задачи спринта
- `/api/backlog` — задачи бэклога
- `/api/issues/[issueKey]` — работа с задачами
- `/api/stories` — работа со стори и эпиками
- `/api/features` — работа с фичами
- `/api/quarterly-plans` — квартальные планы
- `/api/drafts` — драфты задач
- `/api/teams/members` — участники команды

Подробнее: [API_DOCUMENTATION.md](./API_DOCUMENTATION.md)

## База данных

### Основные таблицы

- `sprint_positions` — позиции задач в спринтах
- `task_links` — связи между задачами
- `sprint_comments` — комментарии в спринтах
- `feature_documents` — документы фич
- `feature_document_versions` — версии документов
- `quarterly_plans` — квартальные планы
- `planned_items` — запланированные элементы (эпики, стори, задачи, драфты) с фазами разработки
- `draft_tasks` — драфты задач (еще не созданные в трекере)
- `quarterly_plan_participants` — участники квартального планирования
- `tech_sprints` — техспринты участников команды
- `vacations` — отпуска участников команды

## Ограничения

- **10 рабочих дней** — спринт ограничен 10 рабочими днями
- **3 части дня** — каждый день разделен на 3 части (утро, день, вечер)
- **Один активный спринт** — в планировщике отображается один спринт
- **Одна доска** — работа с одной доской за раз
